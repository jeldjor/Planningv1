# Planning-GJsystems v10.7

Volledige GitHub-klare versie op basis van v10.5-corrected.

Belangrijkste wijziging: Supabase is leidend voor klanten, planning, daginstellingen, afwezigheid en historie. Productieplanning wordt niet meer uit demo- of oude lokale gegevens opgebouwd.

## Installatie

1. Upload alle bestanden uit deze ZIP naar de hoofdmap van de GitHub Pages-repository.
2. Vervang bestaande bestanden met dezelfde naam.
3. `SUPABASE_V10_7_COMPLETE.sql` is alleen nodig wanneer de v10.6-correctie nog niet succesvol in Supabase is uitgevoerd.
4. Open de app opnieuw en log in als Kim.
5. Importeer het klantenbestand op de laptop en wacht op de melding dat de Supabase-import gereed is.
6. Genereer daarna de planning. De iPhone leest vervolgens dezelfde planning uit Supabase.

“Voordoen als gebruiker” is in deze versie bewust uitgeschakeld. Elk account opent uitsluitend zijn eigen werkruimte.
